● BEST ON DESKTOP

Use a mouse & keyboard

Frame Warp is a mouse-driven aim demo. The whole point is tracking a moving target with high-frequency mouse motion (pointer lock) — which a touchscreen can't do, so the scene would just sit there.

What you're looking at

Two clocks. The 3D scene is drawn slowly (≈30 FPS) into an off-screen texture from a deliberately stale view direction. Every display refresh, that texture is re-projected toward the freshest mouse input — so the view you see is at most one display frame old.

The reprojection

sampleUV = uGuard + (vUv + uDelta) · uScale

  • uDelta — view rotation since render, in display-FOV units (Δu = −Δyaw / fovₓ, Δv = Δpitch / fovᵧ)
  • uScale = 1 − 2·guard, uGuard = guard — the guard-band crop that gives the warp real pixels to pull from

The latency it hides

View-direction latency = now − (input time the shown frame used). Warp off ≈ injected lag + frame wait (~150 ms). Warp on ≈ one display interval (~16 ms at 60 Hz). The chart plots both; the gap is the win.
